## TL;DR

Cut `canopy-website` (an Astro+Tailwind v4 marketing site, *not* React) over from a hand-mirrored Tailwind `@theme` block to the published Style-Dictionary-generated `@canopy-ag/react-ui/tokens.css`. Four files change in canopy-website: `package.json` (add dep), `package-lock.json` (regenerate), `src/styles/global.css` (swap `@theme {…}` → `@import "@canopy-ag/react-ui/tokens.css";`), and a new root `.npmrc` that points the `@canopy-ag` scope at GitHub Packages and auths via `${NODE_AUTH_TOKEN}`. README gains one onboarding line. Vercel needs `NODE_AUTH_TOKEN` set out-of-band on the project (consumer classic PAT with `read:packages` on the `canopy-ag` org). First Vercel preview build verifies React stays out of the bundle — the peer-optional design from s1 should make `@canopy-ag/react-ui/tokens.css` resolve to a pure-CSS subpath with no JS pulled in.

## Problem

The current `canopy-website/src/styles/global.css` opens with:

```css
@import "tailwindcss";

@theme {
  --color-canopy-green: #22C55E;
  --color-canopy-green-light: #4ADE80;
  --color-canopy-blue: #00D4FF;
  --color-canopy-blue-dark: #0099CC;
  --color-canopy-dark: #0B1120;
  --color-canopy-darker: #070F1A;
  --color-canopy-card: #111D2F;
}
```

That `@theme` block is a hand-copied mirror of values that now live canonically in `react/ui/src/tokens/tokens.json` (in canopy-roost) and ship via the published `@canopy-ag/react-ui` npm package (epic s1–s5). With s4 now wiring GitHub Packages publish, the cutover from "two parallel sources of truth" to "import the package's CSS" is unblocked.

The unusual angle here vs. the in-monorepo cutover (s5): canopy-website is **not** a React app. It's Astro 5 + Tailwind v4 + a tiny amount of optional React (for embedded interactives). When we add `@canopy-ag/react-ui` to its deps, we must verify React stays out of the production bundle — the package's `peerDependenciesMeta.{react,react-dom}.optional: true` from s1 is what makes this safe. If Vite's tree-shaking proves inadequate, the documented fallback is to drop `optional: true` upstream and require consumers to declare react as a peer.

## Approach

Three in-PR edits + one new file + one README line + one out-of-PR operator action (Vercel env var).

**Edit 1 — `canopy-website/package.json`.** Add `"@canopy-ag/react-ui": "^0.0.2"` (or whatever version s4's release.yml dispatch publishes) to `dependencies`. No other manifest changes; existing `react`/`react-dom` deps stay (they're peer-deps for `@canopy-ag/react-ui` and `canopy-website` already uses them for its `@astrojs/react` interactives).

**Edit 2 — `canopy-website/.npmrc` (NEW at repo root).** Three lines, no secret material:

```
@canopy-ag:registry=https://npm.pkg.github.com/
//npm.pkg.github.com/:_authToken=${NODE_AUTH_TOKEN}
always-auth=true
```

Mirrors `canopy-roost`'s root `.npmrc` (added in s4 commit `c0fe4427`). `${NODE_AUTH_TOKEN}` is resolved at npm-execution time from the env var; CI/Vercel sets it from a classic PAT, dev sets it locally for `npm install`.

**Edit 3 — `canopy-website/src/styles/global.css`.** Replace lines 1–11 (the `@import "tailwindcss";` + `@theme {…}` block) with:

```css
@import "tailwindcss";
@import "@canopy-ag/react-ui/tokens.css";
```

Everything below line 11 — body styles, `.glass`, `.glow`, `.prose`, `.btn-*`, `.gradient-text`, `.nav-link`, etc. — stays unchanged. Those use hardcoded hex values (`#22C55E`, `#00D4FF`, etc.) directly, not `var(--canopy-*)` references, so the deletion of the `@theme { --color-canopy-* }` block doesn't break any of them. (Future refactor opportunity: convert hardcoded hex to `var(--canopy-*)` references for consistency. Out of scope for this story — keeps the visual-diff-zero acceptance gate clean.)

**Edit 4 — `canopy-website/package-lock.json`.** Regenerated by `NODE_AUTH_TOKEN=<dev PAT> npm install`. The added dep + transitive resolution (the package has zero direct deps; `peerDependencies` are advisory) lands as a small lockfile delta. Commit verbatim.

**Edit 5 — `canopy-website/README.md`.** Add one onboarding line under "Getting Started":

```
> Local install: export `NODE_AUTH_TOKEN=<your classic PAT with read:packages on canopy-ag>`
> before `npm install` so npm can fetch `@canopy-ag/*` packages from GitHub Packages.
```

**Operator action (out-of-PR — surface in PR description as required before merge):**

1. Provision a classic PAT on the `canopy-ag` org with **only** `read:packages` scope.
2. Set Vercel project env var `NODE_AUTH_TOKEN` = that PAT on canopy-website.

Without step 2, the Vercel build will 401/403 on `npm install`. The PR description must call this out explicitly; `/story-pr-review` and `/story-qa-verify` will resurface it.

### Naming note: `--color-canopy-*` vs `--canopy-*`

The current hand-written tokens are named `--color-canopy-green` (with `--color-` prefix — Tailwind v4's convention for registering color utilities like `bg-canopy-green`). The published `@canopy-ag/react-ui/tokens.css` uses `--canopy-green` (no `color-` prefix). Spot-check across `canopy-website/src` confirms **zero** uses of utility classes like `bg-canopy-*` / `text-canopy-*` / `border-canopy-*` — so the prefix difference has no surface-area impact. The `@theme {}` block in the imported file still registers the vars; only the utility-class auto-generation differs, and we don't use those utilities anywhere.

### Data model / API / package deps delta

- **Data model**: none.
- **API**: none (canopy-website is a static marketing site).
- **Package deps**: +1 (`@canopy-ag/react-ui`). The lockfile delta should be small because the package externalizes React.

### UI changes

Zero intended visual delta. The acceptance gate is "renders with brand colors visually unchanged from main" on the Vercel preview.

## Risks & Open Questions

- **`@canopy-ag/react-ui` may not yet be on the registry.** s4's PR #359 merged at 2026-05-18 10:42 UTC, but the publish step is gated on a `react-ui@*` tag, which is only created when `release.yml` (workflow_dispatch) fires with `release_mode=true`. As of this story's plan write, the autopilot supervisor has dispatched `release.yml` with `projects=react-ui, version_bump=patch` (run 26029835861). If the run fails, the npm install step below halts class-C and the operator must investigate.
- **React-in-bundle risk.** The peer-optional pattern (s1) hinges on `peerDependenciesMeta.react.optional: true` + Vite tree-shaking. canopy-website uses `@astrojs/react` for islands, so React IS already in the bundle for those islands — but it must NOT be pulled into the *base* bundle by `@canopy-ag/react-ui/tokens.css`. The `tokens.css` subpath should be a pure CSS file that the Vite/Astro pipeline imports inline; it has no JS at all. If, post-build, `dist/` reveals React being imported by the global stylesheet path (extremely unlikely), the documented fallback is to drop `optional: true` upstream.
- **`.npmrc` token resolution on Vercel.** Vercel passes env vars to the build container before `npm install`. The committed `.npmrc` resolves `${NODE_AUTH_TOKEN}` at npm runtime. If Vercel's npm install runs in a context where env-var substitution in `.npmrc` doesn't happen (rare; documented for some private registries), the fallback is `NPM_RC` env var with the full file content. Test on first preview build.
- **Classic PAT scope.** `read:packages` only — never `write:packages`. Limits blast radius if the token leaks. Consumer-side; not the same PAT canopy-roost's release uses.
- **`local-only` mode for the parent epic.** This story's parent epic is local-only (no Notion page). All Notion writes by sub-skills are skipped automatically (per each sub-skill's local-only branch).

## Implementation Outline

Ordered steps. Each can be a separate commit; squash on merge.

1. **Verify package availability.** From canopy-website worktree (NOT canopy-roost), `NODE_AUTH_TOKEN=<dev PAT> npm view @canopy-ag/react-ui versions --registry=https://npm.pkg.github.com/`. Confirm `0.0.2` (or whatever s4's release workflow published) is listed. If the package is missing, HALT — the release workflow either hasn't completed or failed; surface to the operator.
2. **Add `.npmrc` at repo root** (3 lines as above). `cat .npmrc` to verify.
3. **Edit `package.json`** to add `"@canopy-ag/react-ui": "^<version>"` in `dependencies` (alphabetical order — between `@astrojs/vercel` and `@tailwindcss/vite`).
4. **Run `NODE_AUTH_TOKEN=<dev PAT> npm install`.** Expect success: lockfile updates, `node_modules/@canopy-ag/react-ui/` appears. If 401/403, the PAT lacks `read:packages` or the package isn't published yet — halt.
5. **Verify the resolved CSS file ships.** `ls node_modules/@canopy-ag/react-ui/tokens.css` should exist and contain the `@theme { --canopy-* }` block matching the source.
6. **Edit `src/styles/global.css`.** Remove lines 3–11 (`@theme {…}` block + the surrounding blank lines). Add `@import "@canopy-ag/react-ui/tokens.css";` immediately after `@import "tailwindcss";`. Verify no other code reads the deleted `--color-canopy-*` vars (grep -rE 'var\(--color-canopy' src/ — expect zero hits).
7. **Local smoke test.** `npm run dev` and open the site in a browser. Eyeball the hero, cards, prose, buttons — brand colors should be identical to main. Tear down dev server.
8. **Local build.** `npm run build` (which runs `astro check && astro build`). Expect green. Then `ls dist/` and inspect bundle sizes — look for any new React entry. The Astro build produces both static HTML and (for islands) JS bundles; the tokens.css should land in the static-CSS bundle, NOT pull React.
9. **Edit `README.md`** to add the NODE_AUTH_TOKEN onboarding line under "Getting Started".
10. **Commit + push + open PR.** PR description MUST include:
    - Reference to parent epic + s4 dependency satisfied.
    - **Operator actions block** (Vercel `NODE_AUTH_TOKEN` env var + PAT provisioning) called out as REQUIRED before merge.
    - Reminder that Vercel preview build is the gating QA: it must (a) complete `npm install` without 401/403, (b) deploy, and (c) preview URL renders correctly.
11. **Watch Vercel preview build.** First-time auth issues may surface here. If 401/403, operator must verify Vercel env var. If green, manually open preview URL and eyeball brand colors.
12. **Bundle inspection.** From the merged build (or the preview build's `dist/`), confirm React is NOT in the global JS bundle. `npx vite-bundle-visualizer` (or just `find dist -name '*.js' -exec grep -l react {} \;` filtered to exclude expected island bundles). If React leaked, document and follow the fallback per the brief.
